# Бизнес-логика калькулятора

## Общее назначение калькулятора

Калькулятор предназначен для расчета мультимодальной логистической ставки по сделке CRM. Результат расчета должен включать:

- себестоимость
- маржу
- цену клиенту
- детализацию по тарифным строкам
- связку результата с конкретной сделкой Битрикс24

Источником расчета являются справочники, тарифные таблицы, параметры маршрута и пользовательский ввод в карточке сделки.

## Типы маршрутов

### `KLD_OUT`

Маршрут с логикой отправки из схемы, связанной с Калининградом. Точный бизнес-смысл направления должен быть подтвержден.

- `NEEDS_CONFIRMATION`: точное определение стартовой и конечной точки маршрута
- `NEEDS_CONFIRMATION`: какие этапы являются обязательными для этого направления

### `KLD_IN`

Маршрут с логикой входящего направления, связанного с Калининградом. Точный бизнес-смысл направления должен быть подтвержден.

- `NEEDS_CONFIRMATION`: точное определение стартовой и конечной точки маршрута
- `NEEDS_CONFIRMATION`: какие этапы являются обязательными для этого направления

## Этапы расчета

### `FIRST_MILE`

Первичный участок маршрута до основной магистрали или порта.

Предположительно учитывает:

- точку загрузки
- расстояние
- тип и статус контейнера
- вес
- признаки `is_heavy`, `is_dangerous`

### `MAINLINE`

Основной магистральный участок. В legacy-коде встречается разделение на морское плечо и магистральные тарифы, поэтому в новой версии нужно сохранить абстракцию основного этапа, но внутренне поддержать разные тарифные источники.

- `NEEDS_CONFIRMATION`: использовать ли в UI единый этап `MAINLINE` или явно показывать морской сегмент отдельно

### `LAST_MILE`

Финальный участок маршрута до точки выгрузки/доставки.

Предположительно учитывает:

- точку выгрузки
- расстояние
- признаки `is_dangerous`, `is_genset`
- контейнерные параметры

- `id`
- `region_scope` или тип таблицы: `KALININGRAD` / `MOSCOW` / `SPB`

- `min_km`
- `max_km`

- `updated_at`

- `valid_from`
- `min_km`
- `max_km`

## Общий алгоритм расчета

1. Получить входные параметры.
2. Определить тип маршрута.
3. Разбить маршрут на этапы.
4. Найти применимые тарифы.
5. Рассчитать себестоимость.
6. Рассчитать маржу.
7. Рассчитать цену клиенту.
8. Сохранить расчет.
9. Передать результат в Битрикс24.

### Детализация алгоритма

#### 1. Получить входные параметры

Минимально ожидаемые входные параметры:

- `dealId`
- тип маршрута
- локации отправления и назначения
- тип контейнера
- статус контейнера
- категория морского фрахта при наличии морского плеча
- расстояния по автоэтапам
- дополнительные признаки: `is_heavy`, `is_dangerous`, `is_genset`
- контрагент
- параметры хранения, аренды и дополнительных услуг

#### 2. Определить тип маршрута

На основе выбора пользователя или шаблона маршрута определяется `KLD_OUT` либо `KLD_IN`.

#### 3. Разбить маршрут на этапы

Маршрут разбивается минимум на:

- `FIRST_MILE`
- `MAINLINE`
- `LAST_MILE`

Внутри `MAINLINE` могут применяться:

- магистральные тарифы
- морской фрахт
- ПРР
- хранение
- аренда контейнера
- реф-подключение

#### 4. Найти применимые тарифы

Для каждого этапа и дополнительного блока система:

- определяет нужную тарифную таблицу
- строит ключ поиска
- ищет запись тарифа
- фиксирует идентификатор найденного тарифа в составе snapshot/line items

#### 8. Сохранить расчет

Сохраняются:

- шапка расчета

- snapshot входных данных

После сохранения backend:

- возвращает frontend подтверждение записи
